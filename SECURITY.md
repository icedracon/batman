# Security Policy

Batman is security research tooling for private Ethereum upgrade-readiness testing.

## Supported scope

- Local/private devnets.
- Synthetic fixtures generated by this repository.
- Publicly documented Ethereum fork specifications and client builds.

Do not use Batman against mainnet, public RPC providers, third-party infrastructure,
or systems you do not control.

## Reporting suspected client issues

If Batman output suggests a real execution-client divergence:

1. Reproduce it on a private devnet.
2. Preserve the trace, client versions, image digests, and exact command line.
3. Do not publish exploit details or suspected client-vulnerability details publicly.
4. Contact the affected client team or the relevant Ethereum disclosure channel.

Synthetic fixtures are controls. They are useful for testing the detector, but they
are not evidence of a real client bug until replaced by live client outputs.

## Secrets

Never commit JWT secrets, validator keys, prefunded devnet private keys, or local
endpoint files. The repository ignores generated devnet runtime files by default.
